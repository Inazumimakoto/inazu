# MAS TODO

## 概要

- 目標は、`mas.inazu.me` で遊べるローカル駆動の MAS を作ること
- ユーザーが題材を入力すると、複数エージェントがその題材について議論する
- エージェントは 3D 空間上に存在し、近づいて会話し、吹き出しを出す
- 最終的には 8 エージェント前後まで拡張したい

## 今の前提

- フロントエンドは `three.js`
- バックエンドは既存の Node.js / Express を拡張する
- 推論は `llama.cpp` を使う想定
- 長期記憶は持たない
- 永続化はしない
- ブラウザをリロードしたら状態はすべて消えてよい
- 最初から分散 A2A にはしない
- まずは 1 つのローカルシミュレータ内で複数エージェントを動かす

## 非目標

- 初期段階で 8 体を安定運用すること
- 長期記憶、ベクタ DB、プロフィール学習
- 本格的なマルチプロセス分散構成
- Cloudflare Tunnel や本番公開設定の先行対応

## 想定アーキテクチャ

```text
Browser (three.js)
  - 3D 表示
  - 題材入力 UI
  - 吹き出し / ログ / 状態表示
          |
          | WebSocket or SSE
          v
Node.js Orchestrator
  - エージェント状態管理
  - 会話順制御
  - 距離判定
  - イベント生成
          |
          | HTTP
          v

  - 発話が必要な時だけ推論
```

## MVP 仕様

### ユーザー体験

- ユーザーが題材を 1 つ入力する
- 3 体のエージェントが空間内に表示される
- 話す時は相手に近づく
- 発話時に吹き出しが表示される
- 数ターン議論が続く
- 会話ログが画面に残る
- リロードすると全状態が消える

### エージェント最小仕様

各エージェントは最低限これを持つ。

- `id`
- `name`
- `role`
- `personality`
- `position`
- `targetPosition`
- `state`
- `lastUtterance`

初期ロール案:

- 肯定派
- 懐疑派
- 要約役

### ワールドルール最小仕様

- 一定 tick ごとに状態更新する
- 次に話すエージェントを 1 体だけ選ぶ
- 発話対象のエージェントを 1 体選ぶ
- 距離が遠ければ近づく
- 距離が閾値以内なら会話イベントを発火する
- 発話後は少し離れるか待機する

### 初期会話仕様

- 最初は LLM なしのダミー会話でよい
- ダミー会話でも、入力された題材を文面に差し込む
- LLM 接続後は、エージェントごとの `role` と `personality` を prompt に反映する
